¿Qué es cada cosa en tu proyecto?
----------------------------------
• carpeta "sitio-web"     → Esta página de venta / catálogo (HTML)
• carpeta "v0-editorial-control-station" → Panel de autores, Vault, blog (Next.js)

El archivo IMPORTANTE para el sitio web es:  sitio-web/config.js
Ahí van SITE_CANONICAL, APP_BASE (panel), MAKE_CONTACT_WEBHOOK_URL (formulario → Make).


PASO A — Publicar el sitio web (HTML estático)
----------------------------------------------
Recomendado: Vercel, proyecto framework "Other", root del repo, dominio tipo
https://soberania-editorial.vercel.app → esa URL va en config.js como SITE_CANONICAL
(sin barra final). Cada push a main redeploya solo.

Alternativa: GitHub Pages (Settings → Pages desde branch main). Si el sitio “oficial”
sigue en Pages, SITE_CANONICAL debe ser la URL de Pages (con o sin subpath, consistente).


PASO B — La app del panel (Vercel) — ya la tenés o la creás gratis
------------------------------------------------------------------
1) El proyecto Next está en v0-editorial-control-station.
2) En vercel.com (cuenta gratis): Add New → Project → importás el repo.
3) Vercel te da un link tipo: https://algo.vercel.app
4) Ese link lo pegás en sitio-web/config.js en APP_BASE.
5) Subís config.js otra vez al repo del sitio web.

Formulario de contacto (simple):
• La URL del webhook de Make va en sitio-web/config.js → MAKE_CONTACT_WEBHOOK_URL.
• No hace falta configurar el contacto en Vercel del catálogo: el navegador envía directo a Make.
• Si en Vercel habías cargado variables solo para el contacto, podés borrarlas; no se usan con este esquema.


PASO C — Google (gratis) — cuando tengas la URL final del sitio web
-------------------------------------------------------------------
1) Entrá a: https://search.google.com/search-console
2) Agregar propiedad → Prefijo de URL → pegá la misma URL que SITE_CANONICAL.
3) Verificá (Google te explica con una etiqueta o un archivo).
4) Menú "Sitemaps": si tu sitio es solo HTML, Google igual rastrea por enlaces;
   la app Next aparte tiene /sitemap.xml en Vercel.


¿Qué ya está hecho en el código?
---------------------------------
• index.html — navegación por “ventanas” (inicio, contenidos, informes, vault) + vidriera y catálogo abajo
• informes/ — los tres HTML del informe (copiados al repo)
• media/ — podcast .m4a (archivo grande; GitHub acepta hasta ~100 MB por archivo)
• js/site.js — lógica del menú y reproductor
• config.js — URLs, podcast, lista de reels con miniaturas
• Términos: terminos.html


Variables de entorno en Vercel (panel Next, desarrollo avanzado)
-----------------------------------------------------------------
Settings → Environment Variables. Para desarrollo local del panel Next, ver
v0-editorial-control-station/.env.example (Supabase, Mercado Pago, etc.).
El contacto del sitio estático no usa Vercel.
